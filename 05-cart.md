# 장바구니 (Cart) 도메인

> 계층: 일반 도메인 · 투자: 돌아가게만
> 주문 직전의 임시 상태. Redis 기반.

---

## 이 도메인의 책임

사기 전 담아두는 곳. 확정 전이라 언제든 바뀔 수 있는 임시 데이터.

---

## 요구사항 (무엇을)

### 담기
- SKU 단위로 담음 (수량 포함)
- 같은 SKU 다시 담으면 수량 합산
- 회원 장바구니는 Redis에 보관

### 조회/수정
- 담긴 항목 목록 + 현재 가격 (카탈로그에서 조회해 표시)
- 수량 변경, 항목 삭제, 전체 비우기

### 주문 연계
- 선택 항목으로 주문 생성
- 주문 성공 시 해당 항목 장바구니에서 제거

---

## 범위 밖 (지금 안 함)

- 비회원 장바구니 → 회원 병합 (회원 장바구니만 다룬다)
- 장바구니 재고 홀드 (담아도 재고 예약 안 함 — 주문 시점에만 차감)
- 찜/위시리스트

## 결정해야 할 것 (어떻게 — 네가 방어하며 정한다)

1. 왜 장바구니를 RDB가 아니라 Redis에? 이 데이터의 특성이 뭐길래?
2. 장바구니에 담을 때 가격을 저장할까, 조회 시점에 카탈로그에서 가져올까? (가격이 바뀌면?)
3. Redis에 어떤 자료구조로 담을까? (Hash / String+JSON) 트레이드오프는?
4. 장바구니에 담긴 SKU가 그 사이 품절/삭제되면 조회 시 어떻게 처리?
