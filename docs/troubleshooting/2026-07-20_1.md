## 문제
- 증상: schema.sql로 sql 테이블 생성하였지만 테이블이 없다고 뜨는 문제
- org.h2.jdbc.JdbcSQLSyntaxErrorException: Table "members" not found (candidates are: "MEMBERS"); SQL statement:
  SELECT "members"."member_id" AS "member_id", "members"."name" AS "name", "members"."email" AS "email", "members"."password_hash" AS "password_hash" FROM "members" WHERE "members"."email" = ? [42103-240]

## 원인
- h2-console에서는 대소문자 가리지 않고 사용할 수 있다. select * from members, SELECT * FROM MEMBERS 모두 가능
- 하지만 쿼리를 보면 쌍따옴표가 붙어서 나간다. 쌍따옴표가 붙으면 정확히 적힌 표기대로 테이블을 조회하기 때문에 이를 구분하는게 중요하다.

## 해결
- jdbc:h2:mem:testdb;DATABASE_TO_LOWER=TRUE 로 datasource를 지정해주어 소문자를 허용하였다.
- 도메인에 적은 테이블 명은 소문자를 유지하기 위함이였다. @Table("members")
