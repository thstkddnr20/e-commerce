## 왜 필요한가

## 대가는
프로젝트 실행 불가

## 내 프로젝트엔
확실히 null 불가 객체가 들어오는 곳에는 !! 문법으로 강제, null이 가능한 곳은 null 들어올 경우 잡아서 예외처리

## 헷갈렸던 것
- 변수뒤에 ?를 붙여 null 가능 선언하는 것 <- ?가 붙냐 안붙냐에 따라 타입 자체가 달라지는거임
- nullable을 그대로 사용할 시 컴파일 단계에서 막힘
- ?. 을 사용하여 safe call, null아니면 불러오고, null일시 null
- ?: 을 사용하여 null일 시 예외처리 또는 대체 값 지정 가능
- !! 을 사용하여 null 아니라고 우기기
- as? 로 safe cast하기 cast 안될경우 null 반환
